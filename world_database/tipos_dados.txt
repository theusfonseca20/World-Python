Os tipos de dados definem a natureza dos valores que podem ser armazenados em uma coluna de uma tabela

* Escolher o tipo de dado é crucial para a eficiência, integridade e otimização do banco de dados
* Existem muita discussão em relação a quais tipos de dados são melhores ou piores em cada caso

Alguns exemplos:
* Dados numéricos:
    1- SMALLINT: Inteiro de 2 bytes, ou seja, tem um campo de alcance muito baixo
    2- INT/INTEGER: inteiro de 4 bytes
    3- BIGINT: inteiro de 8 bytes (números maiores)
    4- FLOAT: Número com ponto flutuante de precisão simples
    5- DOUBLE: Número com ponto flutuante de precisão dupla
    6- DECIMAL/NUMERIC: Número de precisão fixa, útil para valores monetários

* Dados de texto:
    1- CHAR(n): cadeia de caracteres de comprimento fixo
    2- VARCHAR(n): cadeia de caracteres de comprimento variável
    3- TEXT: cadeia de caracteres muito grande
